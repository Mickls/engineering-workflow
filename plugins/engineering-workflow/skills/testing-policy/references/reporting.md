# 测试汇报

## 失败报告

测试失败影响下一步行动时，报告：

- 复现命令。
- 失败测试名和断言。
- 被测入口或调用链层级。
- 失败分类：生产逻辑、测试不匹配、环境/不稳定、未知。
- 支撑该分类的证据。
- 下一步：调整测试、请求允许修改生产逻辑，或继续调查。

## 测试结果

汇报测试结果时包含：

- 被证明的验收场景 ID 和用户可见行为。
- 运行的命令。
- 结果。
- 失败测试，如有。
- 失败属于生产逻辑、测试不匹配还是环境问题。
- 被测入口层级：真实入口、public contract、service/package、内部 helper。
- 为什么该入口足以证明目标行为。
- 实际返回、状态或副作用与编码前验收场景是否一致。
- 剩余风险。
