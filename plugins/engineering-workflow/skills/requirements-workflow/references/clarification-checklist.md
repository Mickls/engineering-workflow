# 澄清问题清单

优先通过代码、测试和文档确认；只有会影响实现路径且无法自行确认时才问用户。

## 需求语义

- 用户说的术语在项目中是否已有 canonical term？
- 当前行为和目标行为差异是什么？
- 哪些行为必须保留？

## 边界条件

- existing / non-existing 是否都要覆盖？
- 手动入口和自动入口语义是否不同？
- root 内外、include/exclude、权限、租户是否影响行为？
- 失败、重试、并发、重复调用是否有特殊语义？

## 数据和兼容性

- 是否影响旧数据？
- 是否影响旧配置或旧 API？
- 是否需要迁移、回填或兼容读取？

## 验证

- 用户最关心哪个完整链路？
- 哪些结果必须自动测试？
- 哪些只能手动验证？
